Layer — 水平分层视图

本栈 7 层,从物理算力一路抬到模型层。不是 3 层、不是 10 层——每一层都对应一个 "下一层不够"的具体问题。下面的删层思想实验:每删一层都会失去一个真实能力。

删层思想实验

每删一层,失去什么能力?这是判断一层"必要"的最直接方法。

删 ② 网络层(tunnel + peer-proxy)

slave 在 NAT 后面则不能被 driver 调用。无法跨家庭/办公网部署 slave。 集群必须全在同一公网 + 同一防火墙策略。

删 ③ 协议层(workspace/agent/task)

没有 workspace 隔离,多团队/多项目共享一台 agentserver 会互相看见任务; 没有 task 抽象,所有调用退化成裸 HTTP RPC,丢失生命周期追踪。

删 ④ 调度层(driver MCP tools + dispatch)

用户的 Claude Code 没有统一的"派任务"接口,要为每个 slave 写一个 MCP; 任务 routing/fanout/wait/resume 全要 caller 自己实现。

删 ⑤ 能力层

slave 的"能做什么"只能靠 chat 模型即兴决定;集群无法对外公告能力清单, driver 无法做容量规划/能力发现/dry-run。

删 ⑥ 应用层(contract + capability journal + humanloop)

任务边界不清(没有 contract 校验),能力增长不持久(没有 journal), 模型中途要人决策时只能"猜"(没有 humanloop pause/resume)。 本栈相对竞品的所有"上层智识贡献"都集中在这一层。
